17 /* Optimized col-major matrix * vector product:
18  * This algorithm processes the matrix per vertical panels,
19  * which are then processed horizontaly per chunck of 8*PacketSize x 1 vertical segments.
20  *
21  * Mixing type logic: C += alpha * A * B
22  * | A | B |alpha| comments
23  * |real |cplx |cplx | no vectorization
24  * |real |cplx |real | alpha is converted to a cplx when calling the run function, no vectorization
25  * |cplx |real |cplx | invalid, the caller has to do tmp: = A * B; C += alpha*tmp
26  * |cplx |real |real | optimal case, vectorization possible via real-cplx mul
27  *
28  * The same reasoning apply for the transposed case.
29  */

203 /* Optimized row-major matrix * vector product:
204  * This algorithm processes 4 rows at onces that allows to both reduce
205  * the number of load/stores of the result by a factor 4 and to reduce
206  * the instruction dependency. Moreover, we know that all bands have the
207  * same alignment pattern.
208  *
209  * Mixing type logic:
210  * - alpha is always a complex (or converted to a complex)
211  * - no vectorization
212  */
